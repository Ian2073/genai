# 舊版檔案策略

這份文件用來定義哪些檔案屬於主線、哪些屬於舊版（legacy）/實驗版，避免未來接手的人改錯檔。

## 主線原則

以下類型屬於主線：

- `chief.py` / `pipeline/`
- `story.py` / `story_core/`
- `image.py`
- `trans.py`
- `voice.py`
- `utils.py`
- `kg.py`

這些檔案是目前應優先閱讀、修改、驗證的路徑。

## 舊版（Legacy）/實驗檔原則

以下類型應視為 legacy 或實驗版，而不是主線：

- 以模型名稱直接命名的大型單檔腳本
  - 例如：`story_qwen35_9b.py`
- `backups/` 內的歷史備份
- `research/` 內的研究型專案與論文分析資料
- 已被新 package/模組取代、但暫時保留供比對的舊流程檔

另外，以下屬於「相容入口」，不建議直接刪除：

- `evaluator.py`（研究評估流程相容入口）

`exllamav2` 相容層目前統一放在 `runtime/exllamav2_shim.py`，不再保留 root 的 `shim.py` 轉接檔。

這些檔案可以保留，但應明確視為：

- 研究用參考
- 實驗記錄
- 舊版行為比對材料

而不是新的功能擴充入口。

## 未來接手建議

若未來需要保留 legacy 檔，建議遵守：

1. 不在 legacy 檔上直接疊加主線功能。
2. 若要沿用其中邏輯，應抽回主線模組，而不是繼續把主線做在 legacy 檔內。
3. 若 legacy 檔仍需保留，至少在檔頭或文件中標註其角色。
4. 若某個 legacy 檔已無比較或研究價值，應移到封存區而不是留在主線入口旁。

## 一句話判斷

如果一個檔案：

- 沒有被 `chief.py` / `pipeline` / `story.py` 主線流程使用
- 主要只是某次模型實驗的變體
- 名稱綁定特定模型或特定時期實作

那它就應該被視為 legacy，而不是主線。
